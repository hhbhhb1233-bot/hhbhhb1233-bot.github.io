环境准备

安装必要的软件和工具

前置条件

  • 一台电脑 (Windows / macOS / Linux 均可)
  • iPhone 或 iPad, 系统版本 >= iOS 17
  • 一根数据线连接手机和电脑
  • 已安装 Python 3.12+
  • 已安装 uv 或 pip3 (任选其一)

Windows 用户: 必须安装 iTunes, 否则无法识别 iOS 设备。

安装 Python

如果你还没有安装 Python, 请前往 python.org 下载安装 (建议 3.13 版本)。安装时请勾选 "Add Python to PATH"。

安装 uv (推荐)

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
curl -LsSf https://astral.sh/uv/install.sh | sh
2

下载工具

获取 iOSRealRun 并安装依赖

克隆项目

git clone https://github.com/BiancoChiu/iOSRealRun-cli-18.git
cd iOSRealRun-cli-18

安装依赖

uv sync
pip3 install -r requirements.txt

安装完成后，项目目录中会有 main.py、config.yaml、HNroute.txt 等文件。

连接设备并运行

将 iPhone 连接电脑, 开始模拟跑步

连接设备

  • 用数据线将 iPhone/iPad 连接到电脑
  • 解锁设备, 如果弹出"信任此电脑", 点击信任
  • 确保只有一台 iOS 设备连接到电脑

运行程序

Windows 必须以管理员身份打开终端 (右键 CMD 或 PowerShell -> 以管理员身份运行)

# 进入项目目录
cd iOSRealRun-cli-18

# 使用 uv 运行
uv run main.py

# 或使用 pip 安装后直接运行
python main.py
# 进入项目目录
cd iOSRealRun-cli-18

# 使用 uv 运行 (需要 sudo)
sudo uv run main.py

# 或使用 pip
sudo python main.py

需要管理员/root 权限是因为需要创建 TUN 网络设备。

注意事项

  • 程序会无限循环模拟跑步, 按 Ctrl+C 终止
  • 请勿直接关闭窗口, 否则 GPS 定位无法恢复
  • 如果定位未恢复, 重启手机即可
  • 如果一直提示"无设备连接" (Windows), 请确保已安装 iTunes

Android 虚拟跑步

安卓手机 / 模拟器 | Android 11+

1

环境准备

下载 FakeGPS 应用并准备运行环境

选择运行方式

前置条件

  • Android 手机, 系统版本 Android 11 及以上
  • 无需 Root
  • 支持小米运动健康、华为运动健康等

真机上不支持 Keep。如需在 Keep 上使用, 请选择模拟器方式。

推荐模拟器

  • BlueStacks 5 (推荐, 选择 Android 11 系统)
  • 雷电模拟器 (Android 11 版本)
  • MuMu 模拟器 12

模拟器上支持 Keep、小米运动健康等 App。模拟器无真实 GPS 硬件, 不会出现定位冲突, 稳定性更好。主流模拟器无需额外配置, 点击开始即可运行。

下载 FakeGPS APK

下载 FakeGPS.apk (5.4MB)

下载后安装到手机或模拟器中

2

设备配置

开启开发者选项并设置模拟位置

开启开发者选项

  1. 打开 设置 > 关于手机
  2. 连续点击 "版本号" 7 次, 直到提示"已进入开发者模式"
  3. 返回设置, 找到 开发者选项 (部分手机在"系统" 或"更多设置"中)

设置模拟位置应用

  1. 在 开发者选项 中找到 "选择模拟位置信息应用"
  2. 选择 FakeGPS

如果列表中没有 FakeGPS, 请确保已安装 APK 且版本为 Android 11+。

推荐设置

  • 使用 悬浮窗 方式前台运行, 后台运行可能不稳定
  • 关闭系统的省电优化 / 后台限制 (针对 FakeGPS)

主流模拟器 (BlueStacks、雷电、MuMu) 无需手动配置开发者选项。安装 APK 后直接打开 FakeGPS, 点击开始即可运行模拟。

模拟器安装 APK

  • 方式一: 直接将 APK 文件拖入模拟器窗口
  • 方式二: 使用模拟器自带的文件管理器安装
  • 方式三: 通过 ADB 命令 adb install FakeGPS.apk

运行模拟

启动 FakeGPS 并开始虚拟跑步

操作步骤

  1. 打开 FakeGPS App, 授予定位权限
  2. App 会自动加载内置的大工 3km 路线; 如需自定义路线, 点击 "选择 KML 文件" 加载你导出的 KML
  3. 调整参数:
    • 速度: 默认 3.0 m/s (建议 2.5 - 4.0 m/s)
    • 圈数: 根据需要调整
    • 广播间隔: 默认 0.3 秒, 一般无需修改
  4. 点击 "开始", FakeGPS 开始注入模拟定位
  5. 切换到运动 App (如 Keep、小米运动健康), 开始跑步记录
  6. 等待模拟完成, 运动 App 会记录完整的跑步轨迹

真机用户: 推荐使用悬浮窗方式前台运行 FakeGPS, 后台运行可能导致模拟中断。

注意事项

  • 模拟结束后, FakeGPS 会自动停止位置注入, 定位恢复正常
  • 如定位未恢复, 关闭 FakeGPS 或重启手机/模拟器即可
  • 速度不宜设置过快 (>5 m/s), 否则可能被运动 App 判定为异常
  • 真机不支持 Keep, 模拟器支持 Keep
